# NPM Publishing Checklist

## Files Included in Package âœ…

The following files will be published to npm:

- `system/**/*` - Core framework code (CLI, compiler, runner, libraries)
- `package.json` - Package metadata
- `README.md` - User documentation
- `CHANGELOG.md` - Version history
- `CONTRIBUTING.md` - Contribution guidelines
- `docs/**/*` - Documentation (except internal dev docs)
- `examples/**/*` - Code examples
- `templates/**/*` - Starter templates for `atom create`

## Files Excluded from Package ðŸ”’

The following files are **NOT** published (protected by .npmignore):

### User Application Code
- `app/` - Example/test application (NOT framework code)
- `public/` - Public assets from example app

### Build Artifacts
- `dist/` - Compiled output
- `out/` - Static site generation output
- `*.map` - Source maps
- `*.min.js`, `*.bundle.js` - Minified bundles

### Security & Internal Code
- `atom-engine` - Obfuscated binary (internal use only)
- `system_locked/` - Obfuscated source code
- `build_engine.js` - Binary build script (internal tool)

### Development Files
- `.vscode/`, `.idea/` - IDE settings
- `jsconfig.json` - Framework's own IDE config
- `tailwind.config.js` - Framework's own Tailwind config
- `postcss.config.js` - Framework's own PostCSS config

### Internal Documentation
- `docs/DEVELOPER_GUIDE.md` - Internal dev guide
- `docs/FRAMEWORK_READINESS_ANALYSIS.md` - Internal analysis
- `docs/PRODUCTION_READINESS.md` - Internal assessment
- `docs/TEST_WEBSITE_PROMPT.md` - Internal test prompt

### Environment & Secrets
- `.env*` - Environment variables
- `*.key`, `*.pem`, `*.cert` - Certificates/keys

### Temporary Files
- `*.log` - Log files
- `*.tmp`, `*.temp`, `*.bak` - Temporary files
- `*.cache` - Cache files

### Git & CI/CD
- `.git/` - Git repository
- `.github/` - GitHub workflows
- `package-lock.json` - Lock file (users will generate their own)

## Verification

Before publishing, verify what will be included:

```bash
npm pack --dry-run
```

This shows exactly what files will be in the published package.
